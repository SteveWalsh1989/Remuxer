# Remuxer Docs

This folder is for project documentation that should be kept with the repository.

Start with the root `README.md` for project setup, common commands, architecture, and current product constraints.

Use `docs/current-functionality.md` for the current behavior inventory: user workflow, queue options, output destinations, preset behavior, runtime execution, and known constraints.

Use `docs/ui-ux-standards.md` as the source of truth for Remuxer's interface quality bar. It defines readability, visual hierarchy, icon button, tooltip, color, and visual review rules for SwiftUI work.

Local planning notes live in `docs/planning/`. That folder is intentionally git ignored so implementation notes, experiments, and decision drafts can stay local until they are ready to become durable project docs.
